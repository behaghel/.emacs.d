* Org Confluence

=org-confluence= is a package-style Org integration for Confluence.  It owns
Org ↔ Confluence Storage Format conversion, page import/publish/sync,
Confluence people resolution, inline comment marker preservation/repair, sync
status reporting, and Confluence-backed =org-comments= operations.

This directory is intentionally shaped as a package boundary inside the Emacs
configuration.  The package should remain extractable: reusable behavior lives
under =packages/org-confluence/=, while site defaults, personal leader keys,
Evil/Bépo bindings, and window/layout policy live in consumer configuration.
In this repository, personal activation lives in:

#+begin_example
modules/interactive/org/confluence.el
#+end_example

* Canonical entry points

The canonical package entrypoint is:

#+begin_src emacs-lisp
(require 'org-confluence)
#+end_src

=org-confluence.el= loads the export backend, command facade, comments backend
integration, people-store support, and mode-level source markers.  Users should
activate buffer-local integration with either:

- =org-confluence-mode= for an already linked Org buffer; or
- =org-confluence-mode-maybe= from =org-mode-hook= to enable only when
  Confluence metadata such as =#+CONFLUENCE_PAGE_ID= is present.

Former short wrapper files such as =api.el=, =commands.el=, and =export.el= are
not part of the package boundary.  Code should require the canonical
=org-confluence-*= modules directly.

* Installation

When used from this repository, no separate installation is required.

When extracted as a package, the intended setup should look like this:

#+begin_src emacs-lisp
(use-package org-confluence
  :straight (:type git :host github :repo "behaghel/org-confluence")
  :commands (org-confluence-mode
             org-confluence-mode-maybe
             org-confluence-publish
             org-confluence-publish-dwim
             org-confluence-pull
             org-confluence-sync-current)
  :hook (org-mode . org-confluence-mode-maybe)
  :init
  (setq org-confluence-api-base-url "https://example.atlassian.net"
        org-confluence-api-default-space "~personal"))
#+end_src

=org-confluence-mode= registers the Confluence =org-comments= backend and, when
=org-confluence-enable-org-comments-mode= is non-nil, enables
=org-comments-mode= for local comment commands and overlays.  The package mode
does not install personal leader keys, Evil bindings, or layout policy.

The mode binds one package-native dispatch command by default:

#+begin_example
C-c C-x C  org-confluence-dispatch
#+end_example

Set =org-confluence-keymap-prefix= to nil before enabling the mode to leave the
mode map without a default binding, or set it to another key sequence and call
=org-confluence-rebuild-mode-map=.

* Workflows

** Export and publish

- =org-confluence-export-to-xhtml= writes Confluence Storage XHTML for the
  current Org buffer or subtree.
- =org-confluence-publish= updates an existing Confluence page identified by
  buffer or subtree metadata.
- =org-confluence-publish-dwim= updates an existing page or creates a page when
  enough space/parent metadata is available.
- =org-confluence-publish-force= bypasses stale remote-version and
  inline-comment preservation checks after confirmation.

When an Org buffer records =#+CONFLUENCE_PAGE_VERSION=, publish checks the
current remote page version before uploading assets or editing the page.  If the
remote page changed since the last recorded sync, normal publish refuses and asks
you to run =org-confluence-sync-page-current= or explicitly force publish.

Publishing uploads local image assets before page update.  When
=org-confluence-publish-preserve-inline-comments= is non-nil, publishing fetches
existing page storage and inline comment metadata so active inline comment
markers can be reinserted into newly exported storage XHTML when safe.

** Import and pull

- =org-confluence-pull= refreshes the current Org file from Confluence storage.
- =org-confluence-pull-to-file= accepts =:include-comments t= to pull page
  comments into the adjacent =.comments.org= sidecar while keeping the main Org
  page free of comment content.
- =org-confluence-pull-child-page= selects a descendant page/folder and pulls it
  beside the current Org file.

Import converts common Confluence constructs into readable Org, including
status macros, panel/callout macros, emojis, tables, lists, dates, inline
comment markers, and user mentions.  Org date timestamps export as Confluence
native =<time datetime="YYYY-MM-DD" />= elements; imported Confluence time
elements become inactive Org dates such as =[2026-07-02]=.

** Page and comment sync

- =org-confluence-sync-page-current= syncs page content only.
- =org-confluence-comments-import= imports footer and inline comments into the
  =org-comments= sidecar.
- =org-confluence-comments-push-current= pushes the current local sidecar
  comment to Confluence.
- =org-confluence-sync-current= performs page sync and then comments sync when
  the page sync is safe.

The =org-comments= backend operation =:sync= is intentionally comments-only.
Full page synchronization remains in =org-confluence-sync.el= and is exposed via
=org-confluence-sync-current=.

** Sync status

=org-confluence-sync-status= opens a read-only report for page, attachment,
people, and comment state.  =org-confluence-mode= can show a compact
source-buffer marker whose activation opens this report.  Optional UI callback
variables let personal configuration reuse surrounding windows without making
the package depend on personal context-panel modules.

* Development

Generate this reference from defcustoms/commands with:

#+begin_src shell
scripts/elisp-package-docs \
  --load packages/org-confluence/org-confluence.el \
  --prefix org-confluence- \
  --command-prefix org-confluence- \
  --out packages/org-confluence/docs/generated
#+end_src

Run the focused package tests with:

#+begin_src shell
devenv -q shell -- env HOME=$PWD emacs --batch -Q \
  -L packages/org-comments \
  -L packages/org-confluence \
  -L lisp \
  -l ert \
  -l packages/org-confluence/test/org-confluence-entrypoints-test.el \
  -l packages/org-confluence/test/org-confluence-comments-backend-test.el \
  -f ert-run-tests-batch-and-exit
#+end_src

  -L . \
  -L packages/org-comments \
  -L packages/org-confluence \
  -L lisp \
  -l ert \
