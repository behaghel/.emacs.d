* Smoke protocol

Use this document as a live end-to-end smoke target for the Google Docs adapter.
It intentionally mixes structure, inline semantics, native Google Docs features,
and Veriff working-doc styling in one collaboration-shaped document.

** Expected validation flow

1. Create or link a Google Doc from this Org buffer.
2. Run a normal push.
3. Run =M-x org-google-docs-push-restyle-current= after style changes.
4. Inspect the Google Doc visually.
5. Pull back into Org, or run the debug pipeline with a prefix argument, and
   verify that generated visual chrome does not become author-owned Org content.

** High-value visual checks

- Body text uses Inter on a white page.
- Title, H1, and H2 use the Veriff teal heading treatment.
- Quotes have a teal left border, pale teal background, and a modest indent.
- Source blocks use monospace code text and the source-block surface.
- Callout labels have emoji labels and a small gap before body content.
- Only the important callout has a visible border.
- Lists inside callouts remain native Google Docs bullets/numbers.
- Captions remain captions, not body paragraphs with lost semantic ownership.
- Footnotes are native Google Docs footnotes.

where important loses its special emphasis.
